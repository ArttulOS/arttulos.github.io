We are thrilled you're considering contributing to ArttulOS! Your expertise and effort are invaluable to the growth and improvement of this project. To ensure a smooth and productive collaboration, please take a moment to review these guidelines.

Core Contribution Principles

  • AI-Assisted Code: We embrace modern tools! Code generated with the help of AI (e.g., GitHub Copilot, ChatGPT) is allowed and welcome. However, you, the contributor, are fully responsible for its correctness, functionality, security, and adherence to all other guidelines outlined here. Thoroughly test and understand any AI-generated code before submitting it.
  • Keep it Technical, Not Political: ArttulOS is a technology project focused on software development. To maintain a neutral and inclusive environment, you must not introduce political discussions, symbols, commentary, or affiliations into any aspect of your contribution. This includes code, comments, commit messages, documentation, and any project-related communication channels.
  • Optimization is Key: Strive for efficiency. Your code should be optimized for performance (CPU, memory usage) and resourcefulness. Think about algorithmic complexity and efficient data structures. If a particular design choice prioritizes readability or simplicity over raw speed for a good reason, please document this trade-off in your comments.
  • Clarity Through Comments: Good code is understandable. Your contributions must be well-commented. Comments should explain the 'why' behind complex logic, non-obvious decisions, and the purpose of functions/modules, not just restate what the code does (if it's self-evident). Public APIs and intricate sections particularly need clear documentation.

File Header Requirements

To maintain clarity, traceability, and proper licensing, all source code files contributed to ArttulOS must begin with a header comment block. This block should contain the following information:

  • Project Name: Clearly state "ArttulOS".
  • File License: Specify the license under which this particular file is distributed (e.g., MIT, GPLv3, Apache 2.0). This should be consistent with the overall project license unless explicitly stated otherwise for that specific file.
  • Contributors: A list of individuals (names or consistent handles) who have made significant contributions to this specific file. Please add your name/handle when you make substantial changes.

Example (syntax will vary by language):

// Project: ArttulOS
// License: MIT (or specific license for this file)
// Contributors:
//   - Your Name/Handle (Year of first contribution)
//   - Another Contributor (Year)

// ... rest of the file code ...

Contributor Responsibilities & Legal Considerations

  • Developer Sign-Off / Certificate of Origin: By submitting your contribution (e.g., via a Pull Request), you are certifying that you have the right to submit it under the project's license and that the work is your own original work, or that you have the necessary rights to contribute it. This is often formalized as a Developer Certificate of Origin (DCO) or similar agreement. Ensure you understand and agree to this principle. Specific DCO procedures, if any, will be detailed in our repository (e.g., commit message sign-offs).
  • Adherence to Third-Party Licenses: It is your critical responsibility to ensure that any third-party code, libraries, assets, or other materials you introduce into the project are compatible with ArttulOS's overall license and that you strictly adhere to all terms and conditions of those third-party licenses.
    • Clearly document any third-party inclusions, their source, and their specific licenses.
    • Do not include code or assets with incompatible licenses. If in doubt, ask!
  • Contact Staff for Clarification: If you have any questions about these guidelines, the contribution process, licensing (yours or third-party), or require clarification on any aspect of ArttulOS development, please do not hesitate to contact a member of the ArttulOS staff or core development team. We are here to help ensure your contribution experience is positive and compliant. (Refer to the project's main README or community channels for contact information).

General Best Practices

  • Code Style: While we are still formalizing a strict style guide, please try to match the style of existing code in the project. Consistency is important.
  • Testing: Ensure your code works as expected. Include tests where appropriate, especially for new features or bug fixes.
  • Commit Messages: Write clear, concise, and descriptive commit messages. Explain what changed and why. Refer to any relevant DCO sign-off procedures if implemented.
  • Pull Requests: Keep pull requests focused on a single issue or feature. Provide a clear description of the changes.

Thank you immensely for dedicating your time and skills to ArttulOS! By adhering to these guidelines, you help us build a robust, efficient, legal, and welcoming project. We truly appreciate your contributions and look forward to your amazing work!
